## CRITICAL: Broken Commit on restore-memory-automation Branch

**KnowledgeSync.hook.ts line 22 imports `./lib/config-loader` — that file does not exist in kai.**

`config-loader.ts` is a KAI_ONLY_FILE (sync-to-kai.sh line 118). It exists in the kai repo, not here. The restore commit (`e8ce68e`) refactored KnowledgeSync to use config-loader but never created the file. Running `bun hooks/KnowledgeSync.hook.ts` fails immediately with `Cannot find module './lib/config-loader'`.

**Fix before merging:** Either:
- (A) Create `hooks/lib/config-loader.ts` in kai with `loadDomainKeywords()` and `loadDomainDescriptions()` that read `config/domains.jsonc`
- (B) Remove the config-loader import and inline the domain loading (read domains.jsonc directly, fall back to builtins)

Option A is better — it establishes the config-loader that C1 and C2 also need, and matches the kai architecture. This should become the FIRST task of v5.1.0 execution (or an amendment to the restore-memory-automation commit).
