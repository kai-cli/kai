# v5.9.0 Release Plan — Power User UX

**Theme:** Keyboard-First Interaction + Lifecycle Visualization
**Target:** v5.9.0 (immediately after v5.8.0 ships)
**Depends on:** v5.8.0 (session lifecycle schema, events array, terminal adapters)

## Overview

Transform the PAI Board from a "look at it" dashboard into a "drive it" power tool. Five features, all implemented in `board.html` with zero backend changes. Total: ~295 lines of JS/CSS additions.

The v5.8 release gives us lifecycle data and events — v5.9 is the visual and interaction payoff.

## Phase 1: Command Palette (Cmd+K)

**Goal:** Universal action launcher with fuzzy search across all tasks, sessions, and actions.

### Behavior

- `Cmd+K` (Mac) / `Ctrl+K` (Linux) opens floating palette overlay
- Fuzzy match against: task titles, session names, project names, actions
- Actions include: "New Task", "Launch Session", "Change Phase → build", "Archive", "Focus Mode"
- `Enter` executes selected item, `Escape` closes
- Recent items shown when palette is empty (last 5 accessed)
- `/` key focuses the existing search box (already wired, line 176)

### Implementation

```
board.html additions:
├── CSS: .palette-overlay, .palette-input, .palette-list, .palette-item
├── JS: openPalette(), closePalette(), fuzzyMatch(query, items)
├── JS: buildPaletteItems() — collects all tasks + sessions + actions into searchable list
├── JS: executePaletteAction(item) — routes to existing functions (createTask, updatePhase, etc.)
└── JS: keydown listener for Cmd+K / Ctrl+K
```

### Palette Item Types

| Type | Source | Action on Enter |
|------|--------|-----------------|
| Task | allItems[] | Expand card + scroll to it |
| Session | sessionItems[] | Expand session card |
| Project | libraryItems[] | Open project (launch adapter) |
| Action: New Task | static | Open new task modal |
| Action: Focus Mode | static | Toggle focus mode |
| Action: Change Phase | contextual (selected task) | Call updatePhase API |
| Action: Launch Session | contextual (selected task) | Call launch API |
| Action: Archive | contextual (selected task) | Call archive API |

### Fuzzy Match Algorithm

Simple substring + character-order matching. No external library needed:
```javascript
function fuzzyMatch(query, text) {
  let qi = 0;
  for (let i = 0; i < text.length && qi < query.length; i++) {
    if (text[i].toLowerCase() === query[qi].toLowerCase()) qi++;
  }
  return qi === query.length;
}
```

### Validation

- Cmd+K opens palette, shows all items when query is empty
- Typing filters results in real-time
- Enter on task item expands that card and scrolls to it
- Escape closes palette cleanly
- Palette doesn't break existing search box behavior

## Phase 2: Keyboard Navigation

**Goal:** Navigate and interact with all cards without a mouse.

### Key Bindings

| Key | Context | Action |
|-----|---------|--------|
| `j` / `↓` | Board | Select next card |
| `k` / `↑` | Board | Select previous card |
| `Enter` | Card selected | Expand/collapse card |
| `Escape` | Card expanded | Collapse card |
| `Tab` | Board | Jump to next section |
| `Shift+Tab` | Board | Jump to previous section |
| `l` | Task card selected | Launch session |
| `p` | Task card expanded | Open phase selector |
| `a` | Task card selected | Archive |
| `F` | Anywhere | Toggle focus mode |
| `/` | Anywhere | Focus search box |
| `?` | Anywhere | Show keyboard shortcut overlay |

### Implementation

```
board.html additions:
├── CSS: .card-selected (highlight ring, e.g., box-shadow: 0 0 0 2px #58a6ff)
├── JS: selectedIndex tracking (number, indexes into flat card list)
├── JS: getVisibleCards() — returns all rendered cards in DOM order
├── JS: selectCard(index) — scrolls into view, adds .card-selected
├── JS: keydown dispatcher — routes keys to actions based on context
└── JS: shortcutOverlay() — renders a ? help modal with all bindings
```

### Navigation Model

Cards are indexed linearly across all sections (sessions → in progress → backlog). `Tab` jumps to the first card of the next section. Selection persists across re-renders (matched by slug).

### Validation

- `j`/`k` visually highlights cards with a blue ring
- Selected card scrolls into view if off-screen
- `Enter` expands, `Escape` collapses
- `Tab` jumps between sections
- Keys don't fire when typing in search box or modal inputs
- `?` shows overlay with all shortcuts

## Phase 4: Focus Mode

**Goal:** Single keypress strips the board to only the current active task. Zero cognitive load.

### Behavior

- Press `F` (or select from command palette) → body gets `.focus-mode` class
- All sections hide except the one containing the currently expanded/selected card
- If no card is selected, focus on the first active session or in-progress task
- Header simplifies: just title + "Press Esc to exit focus"
- Press `Escape` or `F` again → exits focus mode
- Focus mode state NOT persisted (intentionally ephemeral)

### Implementation

```
board.html additions:
├── CSS: body.focus-mode .section { display: none; }
│         body.focus-mode .section.focused { display: block; }
│         body.focus-mode .dashboard { grid-template-columns: 1fr; max-width: 700px; margin: 0 auto; }
│         body.focus-mode .header-stats { display: none; }
│         body.focus-mode .header-actions { display: none; }
│         .focus-hint { display: none; font-size: 11px; color: #484f58; }
│         body.focus-mode .focus-hint { display: inline; }
├── JS: toggleFocusMode() — adds/removes .focus-mode, marks one section as .focused
└── JS: getFocusTarget() — determines which section to focus (expanded card > selected > first active)
```

### Validation

- `F` key toggles focus mode on/off
- Only one section visible in focus mode (centered, wider)
- Header shows exit hint
- `Escape` exits focus mode
- Doesn't interfere with typing in search box or modals (check `document.activeElement`)

## Phase 5: Smart Suggestions

**Goal:** Surface the 3 highest-priority tasks to work on next, based on lifecycle data.

### Suggestion Logic (no AI inference — pure heuristic)

Priority scoring:
```javascript
function scoreSuggestion(item) {
  let score = 0;
  
  // Quick wins: high progress percentage (close to done)
  if (item.total > 0) score += (item.passed / item.total) * 40;
  
  // Recency: worked on recently = higher priority
  const hoursSinceUpdate = (Date.now() - new Date(item.updated).getTime()) / 3600000;
  if (hoursSinceUpdate < 24) score += 30;
  else if (hoursSinceUpdate < 72) score += 15;
  
  // Active phase bonus: execute/verify are closer to done
  if (item.phase === 'verify') score += 20;
  if (item.phase === 'execute') score += 15;
  if (item.phase === 'build') score += 10;
  
  // Effort penalty: larger tasks score slightly lower (prefer quick wins)
  if (item.effort === 'deep' || item.effort === 'comprehensive') score -= 10;
  
  return score;
}
```

### Display

- Renders as a subtle "Suggested" pill above the In Progress section
- Shows top 3 tasks as compact cards with score-based reasoning:
  - "Quick win — 7/8 criteria done"
  - "Recent momentum — worked on 2h ago"  
  - "Nearly verified — in verify phase"
- Clicking a suggestion scrolls to and expands that card
- If fewer than 3 qualifying tasks exist, show fewer (or hide section entirely)

### Implementation

```
board.html additions:
├── CSS: .suggestions { background: #1a1f2e; border-radius: 8px; padding: 8px; margin-bottom: 8px; }
│         .suggestion-chip { font-size: 10px; background: #21262d; padding: 4px 8px; border-radius: 4px; }
│         .suggestion-reason { font-size: 9px; color: #484f58; }
├── JS: getSuggestions(items) — scores and returns top 3
├── JS: renderSuggestions(suggestions) — builds HTML for suggestion bar
└── Integration: called at top of left panel in renderBoard()
```

### Validation

- Suggestions appear above "In Progress" section
- Tasks with highest progress show as "Quick win"
- Recently active tasks appear with "Recent momentum"
- Clicking suggestion expands the task card
- Empty board = no suggestions section rendered
- Suggestions update on each SSE refresh

## Integration Notes

### No Backend Changes Required

All features are pure client-side additions to `board.html`:
- Command palette: reads existing `allItems`, `sessionItems`, `libraryItems` arrays
- Keyboard nav: operates on rendered DOM elements
- Timeline: reads `events` from session/task API response (added by v5.8)
- Focus mode: CSS class toggle
- Suggestions: scores existing `allItems` array

### v5.8 API Response Dependencies

Timeline scrubber requires v5.8's `/api/work` response to include:
```json
{
  "sessions": [{ 
    "events": [...],      // From v5.8 Phase 1
    "lifecycle": {...}    // From v5.8 Phase 1
  }]
}
```

If events are missing (pre-v5.8 sessions), timeline gracefully shows empty state.

### Key Guard: Disable Keys During Input

All keyboard shortcuts MUST check:
```javascript
function shouldHandleKey(e) {
  const tag = document.activeElement?.tagName;
  if (tag === 'INPUT' || tag === 'TEXTAREA' || tag === 'SELECT') return false;
  if (document.querySelector('.modal-overlay.active')) return false;
  if (document.querySelector('.palette-overlay.active') && e.key !== 'Escape') return false;
  return true;
}
```
