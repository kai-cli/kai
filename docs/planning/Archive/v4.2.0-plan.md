## Priority Order

### 1. Atomic writes for state files (Issue #2)
**Impact:** High — data integrity on concurrent hook writes
**Effort:** Low

All JSON state writes should use write-to-tmp-then-rename to prevent corruption when multiple hooks fire concurrently (Stop fires 5 hooks, UserPromptSubmit fires 4).

```typescript
function atomicWriteJSON(path: string, data: unknown): void {
  const tmp = `${path}.${process.pid}.tmp`;
  writeFileSync(tmp, JSON.stringify(data, null, 2));
  renameSync(tmp, path);
}
```

Files to fix: `work.json`, `session-names.json`, `current-work.json`, `last-cleanup.json`, `session-name-cache.sh`, any hook writing JSON state.

Medium-term stretch: replace `events.jsonl`, `work.json`, `session-names.json`, `ratings.jsonl` with `pai.db` via Bun's native `bun:sqlite`. Single-file backup, ACID transactions, SQL analytics.

### 2. Algorithm micro-effort tier (Issue #7)
**Impact:** High — reduces overhead for simple tasks every day
**Effort:** Low-medium

Current Standard tier requires: read 350-line file, create PRD, 7 phase transitions. For simple tasks this ceremony takes longer than the work.

New **Micro** tier:
- Budget: <30s
- ISC: 1–4 (no floor gate)
- No PRD creation
- No voice announcements
- Phases collapsed: Observe+Plan → Execute → Verify (3, not 7)

Also: lazy-load Algorithm file — classify effort tier in CLAUDE.md first, only `Read` the full file for Standard+.

---

### 3. Hook payload validation / Zod schemas (Issue #6)
**Impact:** Medium — prevents silent failures when Claude Code changes payload shapes
**Effort:** Medium

Claude Code already broke the transcript format once (`human` → `user`). Add Zod schemas for all hook payloads:

```typescript
// hooks/lib/payload-schema.ts
export const UserPromptPayload = z.object({
  session_id: z.string(),
  transcript_path: z.string(),
  hook_event_name: z.literal('UserPromptSubmit'),
  prompt: z.string(),
});
```

Single adapter layer for transcript parsing — when Claude Code changes format, fix one file.

---

### 4. Split monolithic settings.json (Issue #5)
**Impact:** Medium — maintainability, reduces blast radius of config errors
**Effort:** Medium-high

Split into `~/.claude/config/` domain files using JSONC (Bun supports natively):
- `identity.jsonc`, `hooks.jsonc`, `permissions.jsonc`, `notifications.jsonc`, `preferences.jsonc`
- Replace `_docs` fields with real comments
- Merge to `settings.json` at startup via `BuildSettings.ts` with Zod validation

---

### 5. pai upgrade CLI (Issue #8)
**Impact:** Medium — safer installs for growing user base
**Effort:** High

`pai upgrade` Bun TypeScript CLI:
- Auto-backup current installation
- Integrity validation via `manifest.json` (file checksums, version requirements)
- Atomic swap (current → .old, staging → current)
- Post-upgrade migration hooks
- Auto-rollback on failure
