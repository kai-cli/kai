## Your Domain — Local Knowledge

This section is generated by `pai setup` based on your `config/domains.jsonc` configuration.
After running `pai setup`, your domain-specific local knowledge paths will appear here.

Until then, add your own paths in this format:

| Topic | Path |
|-------|------|
| Your knowledge base | `~/Projects/Knowledge/INDEX.md` |
| Your project repos | `~/Projects/your-project/` |
| Your GitHub repos | `gh issue list --repo your-org/your-repo` |

## Cross-Project Memory

Knowledge accumulated in one project's memory session is available to all projects via these paths.
Read on-demand when context is needed — not auto-injected.

**Shorthand:** `{MEM}` = `~/.claude/projects/-Users-{username}-Projects-`

Add your project memory paths here as they accumulate. Example:

| Topic | Path |
|-------|------|
| Project index | `{MEM}your-project/memory/MEMORY.md` |
