# Clarify

Use this when the request is promising but underspecified.

## Steps

1. Restate the objective in one paragraph.
2. List explicit requirements.
3. List implied requirements.
4. List constraints, risks, and unknowns.
5. Ask only blocking questions. If reasonable defaults exist, state them and proceed.
6. Produce a readiness decision: `ready-for-spec`, `needs-input`, or `park`.
