# v6.6.0 Features: Enhanced Orchestration

This document describes the three new CrewAI-inspired features added to dev-team.ts in v6.6.0.

## 1. Event-Driven Checkpointing

**Module:** `scripts/lib/checkpoint.ts`

Allows dev-team runs to resume from failure points instead of restarting from scratch.

### Usage

```typescript
import { CheckpointManager } from './lib/checkpoint';

const checkpoint = new CheckpointManager(teamDir);

// Before each phase
checkpoint.save('scope', 'started');

// After successful completion
checkpoint.save('scope', 'completed', pmOutput);

// On failure
checkpoint.save('scope', 'failed');

// Resume capability
if (checkpoint.canResume()) {
  const lastPhase = checkpoint.getLastCompleted();
  const output = checkpoint.getPhaseOutput(lastPhase);
  // Continue from next phase
}

// After successful run
checkpoint.cleanup();
```

### CLI Flag

```bash
bun dev-team.ts --preset bug-fix --issue "..." --resume
```

## 2. Conditional Task Execution

**Module:** `scripts/lib/conditions.ts`

Skip phases that aren't needed based on context (output size, cost, etc.).

### Supported Conditions

- `output.X.length > N` - Skip if phase output is too short/long
- `metrics.X > N` - Skip based on numeric metrics
- `metrics.X == "Y"` - Skip based on string metrics

### Built-in Conditions

- **verify**: Skipped if `output.implement.length < 500` (trivial change)
- **review**: Skipped if `metrics.totalCostUsd < 0.50` (not worth reviewing)

### Usage

```typescript
import { evaluateCondition, getPhaseCondition, PhaseContext } from './lib/conditions';

const context: PhaseContext = {
  output: { implement: devOutput },
  metrics: { totalCostUsd: 0.75 },
};

const condition = getPhaseCondition('verify');
if (condition) {
  const shouldRun = evaluateCondition(condition.condition, context);
  if (!shouldRun) {
    console.log(`Skipping verify: ${condition.skipReason}`);
  }
}
```

### CLI Flag

```bash
bun dev-team.ts --preset bug-fix --issue "..." --strict  # Ignores all conditions
```

## 3. DAG Execution Planner

**Module:** `scripts/lib/dag-planner.ts`

Replace implicit sequential ordering with explicit dependency declarations, enabling parallel execution.

### Usage

```typescript
import { buildExecutionPlan, validateDAG, PhaseNode } from './lib/dag-planner';

// Define phases with dependencies
const phases: PhaseNode[] = [
  { id: 'scope', dependsOn: [] },
  { id: 'implement', dependsOn: ['scope'] },
  { id: 'verify', dependsOn: ['implement'] },
  { id: 'security', dependsOn: ['implement'] },
  { id: 'review', dependsOn: ['verify', 'security'] },
];

// Validate
const validation = validateDAG(phases);
if (!validation.valid) {
  console.error(validation.errors);
}

// Build execution plan
const plan = buildExecutionPlan(phases);
// plan = [
//   { tier: 0, phases: ['scope'] },
//   { tier: 1, phases: ['implement'] },
//   { tier: 2, phases: ['verify', 'security'] },  // Parallel!
//   { tier: 3, phases: ['review'] },
// ]

// Execute tiers
for (const tier of plan) {
  await Promise.all(tier.phases.map(phase => executePhase(phase)));
}
```

### Convert Linear to DAG

```typescript
import { linearToDAG } from './lib/dag-planner';

const phases = linearToDAG(['scope', 'implement', 'verify']);
// Produces sequential DAG for backward compatibility
```

**Current:** Feature modules implemented and tested. Integration into dev-team.ts orchestrator is planned for future update.
